    Key decisions reflected (per user decisions):
    - D-01: Use `stats-organization/github-readme-stats-action@v2` (GitHub Actions wrapper) — NOT the public Vercel instance
    - D-03: Dark theme applied to all cards via `theme=dark` option
    - D-04: Full metrics via three cards: stats (stars, commits, PRs, repos), top-langs (6 languages), streak (contribution graph)
    - OpenCode discretion: daily schedule (midnight UTC), on-push trigger for main branch, workflow_dispatch for manual runs
    - OpenCode discretion: GITHUB_TOKEN used (not PAT) for public repo stats; no `count_private=true` since that would require a PAT with repo scope

    Content:

    ```yaml
    name: Update GitHub Stats
    on:
      schedule:
        - cron: "0 0 * * *"
      workflow_dispatch:
      push:
        branches: [main]

    jobs:
      update:
        runs-on: ubuntu-latest
        permissions:
          contents: write
        steps:
          - uses: actions/checkout@v6

          - name: Generate stats card
            uses: stats-organization/github-readme-stats-action@v2
            with:
              card: stats
              options: username=uz6r&show_icons=true&theme=dark
              path: profile/stats.svg
              token: ${{ secrets.GITHUB_TOKEN }}

          - name: Generate top languages card
            uses: stats-organization/github-readme-stats-action@v2
            with:
              card: top-langs
              options: username=uz6r&layout=compact&langs_count=6&theme=dark
              path: profile/top-langs.svg
              token: ${{ secrets.GITHUB_TOKEN }}

          - name: Generate streak card
            run: |
              curl -sL "https://github-readme-streak-stats.herokuapp.com/?user=uz6r&theme=dark" > profile/streak.svg

          - name: Commit cards
            run: |
              git config user.name "github-actions[bot]"
              git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
              git add profile/*.svg
              git commit -m "chore: update github stats cards" || exit 0
              git push
    ```

    Notes:
    - The `streak` card is fetched via curl since the action does not natively support streak card generation. This is the standard approach used in thousands of profile READMEs.
    - The `|| exit 0` guard prevents workflow failure when there are no changes (zero-diff runs).
    - `actions/checkout@v6` matches the action's own example (latest stable).
    - No secrets are needed — GITHUB_TOKEN is automatically provisioned by GitHub Actions.
  </action>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | E (Elevation of Privilege) | GitHub Actions workflow | mitigate | Use default GITHUB_TOKEN with scoped `contents: write` permission. No PAT or broad-scoped token exposed. |
| T-02-02 | T (Tampering) | External streak SVG fetch | accept | The streak SVG is fetched from a well-known public service. Risk of injection is negligible — the SVG is displayed inline on the profile README and GitHub's CDN sanitizes served content. No sensitive data in the profile repo. |
| T-02-03 | S (Spoofing) | Stats card content | accept | Stats are generated from GitHub's own API via the action. No user-supplied input. Card content reflects real profile data. |
